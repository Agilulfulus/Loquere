Let's Talk About Bubble Sort

11/6/2017

Sorting algorithms have a special place in my heart, they're just so damn nifty and fun to do. The first I learned, and probably the first you'd learn in a compsci class, was bubble sort. It's a terrible algorithm when it comes to speed and power, but it's simple enough that you can understand it in about 5 minutes.

The basic idea is this: you have a list of n items, and for each item as you iterate along the list, you compare it to the previous. If it's less than the previous, you swap the two, while if it's larger (or equal, I suppose), you leave it be. This is done repeatedly until a cycle runs with no swaps. In the worst case scenario, the algorithm will loop once for each item, and in the best case it'll loop once (i.e., you sent it a list that's already solved you dumb shit).

A more programmatic way to look at it, there are two loops, each one going over each element, so O(O(n)) or O(n^2). I have no clue if O(O(n)) is actually a thing but it makes sense to me.

With that worthwhile explanation, how about we have some source code! I'll just start out with some basic pseudocode: 

                
sort(array) {
    for n in array do {
        for i from 1 length of array do {
            if array[i] < array[i - 1] then swap
        }
    }
    return array
}

Simple, right? Pretty clear to see why it's inneffecient; it has to loop over everything repeatedly. If you're a professional don't do a bubble sort.
